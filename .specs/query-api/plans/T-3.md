# T-3: Promote Shared Audit Event Persistence Types

**Refs:** US-1 ([requirements.md](../requirements.md)), [design.md](../design.md) section 3 "Entity placement"
**Depends on:** —
**Est. LOC:** ~180

## Goal

Prepare the codebase for a second feature (the query slice) by moving the
audit event persistence types that today live in ingestion-only packages
into `shared/` sub-packages, with **no behavior change**. After this
task:

- `AuditEvent` (JPA entity) lives at
  `domain/shared/AuditEvent.java`.
- `Outcome` (enum) lives at `domain/shared/Outcome.java`.
- `AuditEventRepository` (Spring Data CRUD) lives at
  `infrastructure/persistence/shared/AuditEventRepository.java`.
- Every prior importer in `api/ingestion/`, `application/ingestion/`,
  `infrastructure/persistence/ingestion/`, and matching tests references
  the new packages.
- The ingestion HTTP contract is **byte-identical** before and after the
  commit; persistence schema is untouched; `AuditEventTest`,
  `IngestEventUseCaseTest`, `IngestionIT`, and `LayeringTest` all
  remain green.

`InvalidAuditEventException` **stays in `domain/ingestion/`** — it is
ingestion-specific (thrown only on write-path validation) and design.md
§3 promotes only `AuditEvent` and `Outcome`. T-1's
`GlobalExceptionHandler` continues to import it from `domain/ingestion/`.

## Approach

- Pure-move refactor. No class renames, no field changes, no annotation
  changes, no Javadoc edits, no formatting "improvements" inside the
  moved files. The only diff inside the three moved files is the
  `package` declaration at line 1.
- ADR-0003 says to promote a type to `shared/` **only when 2+ features
  depend on it**. That moment is now: `application/query/` (introduced
  in T-5+) will read `AuditEvent` and use `Outcome`, and the
  query-side persistence adapter (T-8) will need
  `AuditEventRepository`. Doing the promotion here, as a standalone
  commit, isolates the rename from any new feature code so the diff
  reviews as a mechanical move.
- Default Spring Boot component scanning is in use
  (`@SpringBootApplication` in `AuditLogServiceApplication` at the root
  package `net.sam.ai.engineering.audit`, no `@EntityScan` / no
  `@EnableJpaRepositories`). Both `domain/shared/` and
  `infrastructure/persistence/shared/` are descendants of that root, so
  no Spring config edits are required.
- The `LayeringTest` (ArchUnit) uses AntPath globs `..domain..` and
  `..infrastructure..` — `domain/shared/` and
  `infrastructure/persistence/shared/` match those globs, so layering
  assertions stay green automatically.
- No Flyway migration. The DB table name (`audit_events`) and
  `@Table`/`@Column` mappings on `AuditEvent` are unchanged, so
  `spring.jpa.hibernate.ddl-auto=validate` (enabled in the test profile
  per `ARCHITECTURE.md`) continues to match V2.
- Test-first discipline (Ground rule 6) is preserved by introducing a
  **single failing structural test up front** that asserts the new
  package locations exist. This test is the artifact that documents
  intent and that the moves actually happened; once it goes green, the
  existing behavioral suites (`AuditEventTest`,
  `IngestEventUseCaseTest`, `IngestionIT`, `LayeringTest`) act as the
  no-regression check.

## Step-by-step execution

Test-first per Ground rule 6. Surgical / pure-move per task Notes.

1. **Failing test (DoD #1, #3):** add
   `SharedTypePlacementTest` under
   `src/test/java/.../architecture/`. Use ArchUnit
   (`classes().that().haveSimpleName("AuditEvent").should().resideInAPackage("..domain.shared..")`)
   for `AuditEvent`, `Outcome`, and (with
   `..infrastructure.persistence.shared..`) for
   `AuditEventRepository`. Run `./mvnw test` — fails (types still in
   `..ingestion..`).
2. **Move `Outcome`.** Create
   `domain/shared/Outcome.java` with `package
   net.sam.ai.engineering.audit.domain.shared;`. Body byte-identical to
   the original. Delete `domain/ingestion/Outcome.java`. Compile will
   break in every file that imports it.
3. **Move `AuditEvent`.** Create
   `domain/shared/AuditEvent.java` similarly (package line only).
   Delete `domain/ingestion/AuditEvent.java`.
4. **Move `AuditEventRepository`.** Create
   `infrastructure/persistence/shared/AuditEventRepository.java` with
   `package …infrastructure.persistence.shared;` and `import
   net.sam.ai.engineering.audit.domain.shared.AuditEvent;`. Delete the
   ingestion-package copy.
5. **Move `AuditEventTest`.** Create
   `src/test/java/.../domain/shared/AuditEventTest.java` with the new
   package. Delete the ingestion-package copy. No assertion changes.
6. **Update importers.** Edit the eight files in the **Modified** list
   above. For each, replace
   `net.sam.ai.engineering.audit.domain.ingestion.AuditEvent` →
   `…domain.shared.AuditEvent`,
   `…domain.ingestion.Outcome` → `…domain.shared.Outcome`, and (in
   `JpaEventSink.java` only — it does not import the repo itself, but
   nothing else does either except the repo's own file, already
   handled in step 4). No other edits inside these files.
7. **Compile.** `./mvnw -q compile test-compile`. Confirm zero
   compilation errors. If any remain, an import was missed — search and
   fix; do not introduce new code.
8. **Run touched unit tests (DoD #2):** execute `AuditEventTest`
   (now at its new path) and `IngestEventUseCaseTest`. Both must pass
   without behavioral edits.
9. **Run integration suite (DoD #2, #4):** `./mvnw verify`. `IngestionIT`
   must pass unchanged — no response field, status code, or body
   shape differs. Flyway `validate` confirms the JPA mapping still
   matches V2 (entity class moved, `@Table`/`@Column` annotations
   unchanged).
10. **Re-run `SharedTypePlacementTest` (DoD #1):** green.
11. **Re-run `LayeringTest` (DoD #3):** green — `..domain..` and
    `..infrastructure..` globs cover `..shared..` sub-packages
    automatically; no edit to `LayeringTest` required.
12. **Static checks:** `mvn spotless:apply` then `./mvnw test`. Ensure
    formatter does not "improve" any moved class body — if it does,
    that diff must be reverted to keep this a pure move (or the
    formatter rule was already silently violated; in that case note in
    Risks).
13. **LOC budget check (DoD #5):** `git diff --stat main...HEAD`. Five
    file moves (`git` treats them as delete + add but `--find-renames`
    will collapse them) plus eight one-line import edits plus a small
    structural test should land near the ~180 LOC estimate, well under
    400.

## Risks / open questions

- **Git rename detection.** A pure move shows up cleanly as `R100` in
  `git diff -M` only if file content (other than the `package` line)
  is identical. Resist any Spotless re-formatting inside the moved
  files. If Spotless does rewrite them on `mvn spotless:apply`, that
  means the prior files were already non-compliant — note in the PR
  description but still ship as one commit.
- **Coordination with T-1 (`GlobalExceptionHandler`).** T-1's advice
  imports `InvalidAuditEventException` from `domain/ingestion/`. T-3
  does **not** promote `InvalidAuditEventException` (design §3 names
  only `AuditEvent` and `Outcome`), so T-1's import remains valid
  regardless of merge order. Documented here so reviewers don't expect
  T-1 to change.
- **Coordination with downstream tasks (T-5, T-7, T-8).** Those tasks
  will import `AuditEvent` / `Outcome` from `domain.shared.*` and
  `AuditEventRepository` from `infrastructure.persistence.shared.*`.
  Their plans must reference the new package names; flag any draft
  that still references `domain.ingestion.*` for query-side use.
- **Spring Data repository scanning.** `@SpringBootApplication` at
  `net.sam.ai.engineering.audit` uses default scanning, so moving the
  repository under `infrastructure/persistence/shared/` is picked up
  automatically. Verified by Grep: no `@EntityScan` or
  `@EnableJpaRepositories` exists in the codebase. If a future commit
  adds explicit scanning, that commit (not this one) owns the config
  delta.
- **Hibernate JPA validation.** Class package change does not affect
  `@Table(name = "audit_events")` or column mappings; `ddl-auto=validate`
  matches `V2__create_audit_events_table.sql` byte-exact post-move.
  Covered indirectly by `IngestionIT` (it boots the full context).
- **`InvalidAuditEventException` placement reconsidered?** Out of scope
  here. If a future feature needs to throw or catch it from a
  non-ingestion package, that future task promotes it under ADR-0003;
  do not do it opportunistically in this commit.
