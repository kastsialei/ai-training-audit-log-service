# T-6: Implement Cursor Codec and Filter Fingerprint

**Refs:** US-3 ([requirements.md](../requirements.md)), [design.md](../design.md) section 5 "Invariants" (#2 page-stability, #3 cursor binds to filter set), section 6 "Cursor format" + "Cursor failure modes"
**Glossary:** [../glossary.md](../glossary.md) — canonical names, types, exceptions, problem types
**Depends on:** T-5 (provides `QueryCriteria` value record + shared `Outcome` enum after T-3's promotion)
**Est. LOC:** ~280

## Goal

Add the opaque cursor primitive and its filter-set binding without
touching HTTP or DB code. After this task, the application layer can
encode the `(recorded_at, id)` keyset position of "the last item of the
page just returned" as a single `base64url(JSON{rt,id,fp})` string,
decode it back to a typed `Cursor` record, and reject any cursor that
is structurally invalid or was issued for a different filter set. The
result is consumed by T-7 (use case) and T-8 (DB reader). No HMAC, no
TTL, no HTTP — those are explicitly out of scope per `tasks.md` T-6
and `design.md` §6.

## Approach

- **Layering.** `application/query/` only. Pure Java + Jackson. No
  Spring Web, no JPA, no Spring container. `ObjectMapper` is plumbed
  in via the constructor so the codec is unit-testable without any
  bean wiring (T-7 will inject it; tests build their own). Jackson is
  already used inside `application/ingestion/` (`IngestEventCommand`
  takes `JsonNode`) so this introduces no new dependency on the
  application layer's allowed transitive set.
- **Cursor record.** Exactly as mandated by glossary §2:
  `record Cursor(OffsetDateTime rt, UUID id, String fp)`. Compact
  value type. `fp` is a `String` (16 hex chars) not a `byte[4]` — it
  never leaves the codec in raw form and equality semantics for
  strings are sane.
- **Codec class.** `CursorCodec` exposes exactly three methods per
  glossary §3:
  - `String encode(Cursor cursor)` → `base64url(JSON({"rt":..., "id":..., "fp":...}))`.
  - `Cursor decode(String raw, String expectedFingerprint)` → returns
    a `Cursor` iff base64 decodes, JSON parses, all three fields
    present, types valid, and `c.fp.equals(expectedFingerprint)`.
    Single-call shape: decoding and fingerprint verification are
    fused, so callers cannot accidentally use a structurally-valid
    cursor without the binding check.
  - `String fingerprintFor(QueryCriteria criteria)` → public helper
    that returns the 16-hex-char fingerprint for a given criteria.
    Exists so T-7 can (a) compute `expectedFingerprint` before calling
    `decode`, and (b) set `fp` on freshly minted next-cursors without
    reaching into the package-private `FilterFingerprint` utility.
- **Fingerprint computation.** `FilterFingerprint` is a
  **package-private** utility with one method
  `String compute(QueryCriteria filters) → String` (16 hex chars).
  It is **exposed externally only via** `CursorCodec.fingerprintFor(...)`,
  which simply delegates. T-7 (and any future caller) never imports
  `FilterFingerprint` directly. Algorithm:
  1. Build a `TreeMap<String, Object>` containing only non-null
     substantive filters in their **normalized** form — never a
     `HashMap`, to guarantee iteration order is independent of JVM
     hash seed.
  2. Normalize per design §6:
     - `outcome` → `UPPERCASE` name of the enum (matches `Outcome`
       constants exactly).
     - `from`, `to` → `OffsetDateTime` converted to UTC
       (`.withOffsetSameInstant(ZoneOffset.UTC)`) and formatted as
       ISO-8601 with explicit `+00:00` offset
       (`DateTimeFormatter.ISO_OFFSET_DATE_TIME` on a UTC instant
       renders `Z` — we explicitly render `+00:00` for byte-stability
       across JDK micro-versions; verified by a stability test).
     - `actor`, `resource`, `event_type` → byte-exact strings, no
       trim, no case-fold.
  3. Serialize the `TreeMap` with a Jackson `ObjectMapper`
     configured with
     `SerializationFeature.WRITE_DATES_AS_TIMESTAMPS=false` and
     `SerializationFeature.ORDER_MAP_ENTRIES_BY_KEYS=true`. The
     `TreeMap` already enforces order; the Jackson feature is belt +
     suspenders for any nested map (none today, but future-proof).
     No whitespace (Jackson default).
  4. UTF-8 bytes → `MessageDigest.getInstance("SHA-256")` →
     hex-encode the first 8 bytes (= 16 hex chars). Lowercase hex.
- **Encoded JSON shape** uses snake-ish short keys `rt`, `id`, `fp`
  exactly as specified in design §6. Field names are mirrored in the
  record and in the JSON via Jackson `@JsonProperty` so refactors of
  the record can't silently change the wire format.
- **Base64.** `Base64.getUrlEncoder().withoutPadding()` for encode.
  `Base64.getUrlDecoder()` for decode (which already tolerates absent
  padding). Standard `java.util` — no third-party dep.
- **Typed exception (single class, nested `ProblemKind`).** One
  unchecked exception in `application/query/` per glossary §4:

  ```java
  public final class InvalidCursorException extends RuntimeException {
      public enum ProblemKind { INVALID_CURSOR, CURSOR_FILTER_MISMATCH }
      public ProblemKind kind();
  }
  ```

  - `INVALID_CURSOR` covers structural failures: base64 fail, JSON
    parse fail, missing field, type fail (non-UUID `id`, non-parseable
    `rt`, `fp` not 16 hex chars). Message is the literal string
    `"cursor is malformed"`. The underlying `IOException` /
    `IllegalArgumentException` may be attached as `cause` for log
    correlation; it is never echoed into `getMessage()`.
  - `CURSOR_FILTER_MISMATCH` covers the structurally valid cursor
    whose `fp` does not equal the fingerprint computed from the
    current filters. Message is the literal string `"cursor was
    issued for a different filter set"`. Carries no `Cursor`, no
    `fp`, no `QueryCriteria` reference — only the static message.

  Messages are chosen by `ProblemKind` in the constructor
  (`InvalidCursorException(ProblemKind kind)` and
  `InvalidCursorException(ProblemKind kind, Throwable cause)`), so
  callers can never inject user-supplied text. T-10 maps the two
  kinds to `type` URIs `…/invalid-cursor` and
  `…/cursor-filter-mismatch` respectively via a `switch(kind())`.
  Static, kind-keyed messages keep message-leak assertions trivial:
  `assertThat(exception.getMessage()).doesNotContain(...)` against
  the input cursor and any decoded field value.
- **What this task explicitly does NOT do.**
  - No `Instant`/`String` overloads on `encode`. One canonical shape.
  - No HMAC, no `iat` / `exp` claim. (Design §6: "Not signed".)
  - No HTTP plumbing (T-9). No keyset SQL (T-8). No "≥1 substantive
    filter" rule (T-5 already owns this; codec assumes filters are
    valid).

## Files

**Created (src/main/java/net/sam/ai/engineering/audit/application/query/)**
- `Cursor.java` — `record Cursor(OffsetDateTime rt, UUID id, String fp)`.
  ~15 LOC.
- `CursorCodec.java` — `encode(Cursor)`, `decode(String, String)`,
  and `fingerprintFor(QueryCriteria)`. `ObjectMapper` injected via
  constructor. `fingerprintFor` delegates to package-private
  `FilterFingerprint`, giving T-7 a single public entry point for
  fingerprinting without leaking the utility. ~100 LOC.
- `FilterFingerprint.java` — **package-private** utility with one
  method `String compute(QueryCriteria filters)`. Owns the `TreeMap`
  build + normalize + SHA-256 + hex. Reached from outside the
  package only through `CursorCodec.fingerprintFor(...)`. ~80 LOC.
- `InvalidCursorException.java` — `extends RuntimeException`. Nested
  `ProblemKind { INVALID_CURSOR, CURSOR_FILTER_MISMATCH }`.
  Constructor takes `ProblemKind` (and optional `Throwable cause`)
  and selects a static message keyed by kind. Exposes `kind()`. ~25 LOC.

**Created (src/test/java/net/sam/ai/engineering/audit/application/query/)**
- `CursorCodecTest.java` — round-trip and decode-failure matrix. ~90 LOC.
- `FilterFingerprintTest.java` — canonicalization and stability. ~70 LOC.

**Modified**
- None. T-5's `QueryCriteria` is consumed read-only; no signature
  changes there. The new `InvalidCursorException` is a new file;

## Step-by-step execution

Test-first per Ground rule 6. Each step states the **failing test** then
the **just-enough implementation** to turn it green. All tests are
pure unit; no Testcontainers, no Spring slice.

1. **Failing test (DoD: encode/decode round-trip).** In
   `CursorCodecTest::roundTrip_preservesAllFields`, construct a
   `Cursor` with a fixed `OffsetDateTime` (UTC, with sub-millisecond
   precision e.g. `2026-04-30T14:22:01.123456Z`), a fixed `UUID`, and
   a 16-hex `fp`. `encode` then `decode` with the same expected `fp`.
   Assert equality on all three fields. Run — fails (no codec yet).
2. **Implement `Cursor` record + `CursorCodec.encode`.** Wire Jackson
   `ObjectMapper` (configured with the two `SerializationFeature`
   flags above) through the constructor. `Base64.getUrlEncoder()
   .withoutPadding()`. Implement `decode` minimally for the happy
   path (parse + return; fingerprint check comes in step 5). Re-run —
   green.
3. **Failing test (DoD: fingerprint stability — sorted keys).** In
   `FilterFingerprintTest::stableAcrossEqualLogicalFilters`, build two
   `QueryCriteria` instances where filters are constructed in different
   insertion order (e.g. `(actor=u_42, resource=doc:9821)` vs
   `(resource=doc:9821, actor=u_42)`). Assert `compute(a)` ==
   `compute(b)` byte-exact. Run — fails (`FilterFingerprint` missing).
4. **Implement `FilterFingerprint.compute`** with the `TreeMap` +
   Jackson + SHA-256 + 16-hex pipeline. Re-run — green.
5. **Failing test (DoD: fingerprint stability — `outcome`
   uppercased, `from`/`to` UTC, byte-exact strings).** Three
   sub-assertions in one test method or three small methods:
   - Build filters with `Outcome.DENIED` and compare against
     `compute` of an otherwise-equal filter set; assert the JSON we
     feed SHA-256 contains `"DENIED"` literally (use a test-only
     "expose canonical bytes" helper or assert via prefix of the
     known SHA-256 of a hand-rolled canonical string).
   - Build filters with `from = 2026-04-30T16:22:01+02:00` and
     another with `from = 2026-04-30T14:22:01Z`; assert
     `compute(a) == compute(b)`.
   - Build filters with `actor = "u_42"` and `actor = " u_42 "`;
     assert `compute(a) != compute(b)` (byte-exact, no trim).
   Run — failing sub-assertions point to whichever normalization is
   missing; fill them in.
6. **Failing test (DoD: bad base64).** In
   `CursorCodecTest::decode_rejectsBadBase64`, call
   `codec.decode("!!!not-base64!!!", anyFp)`. Assert
   `InvalidCursorException` is thrown, `e.kind() == INVALID_CURSOR`,
   and that `e.getMessage()` equals exactly `"cursor is malformed"`
   (no leak guarantee). Run — currently throws raw
   `IllegalArgumentException` from `Base64`.
7. **Implement bad-base64 branch:** wrap `Base64.getUrlDecoder()
   .decode(...)` in a `try/catch(IllegalArgumentException)` →
   `throw new InvalidCursorException(INVALID_CURSOR, cause)`. Re-run
   — green.
8. **Failing test (DoD: bad JSON).** Valid base64 of the literal
   bytes `not-json`. Assert `InvalidCursorException` with
   `kind() == INVALID_CURSOR` and same fixed message. Run — currently
   throws raw `JsonProcessingException`.
9. **Implement bad-JSON branch** — catch `IOException` from
   `ObjectMapper.readTree` / `readValue` →
   `throw new InvalidCursorException(INVALID_CURSOR, cause)`. Re-run
   — green.
10. **Failing test (DoD: missing fields).** Encode three cursors with
    one of `rt`/`id`/`fp` omitted from the JSON (hand-build the JSON
    bytes, base64url, call `decode`). Each must throw
    `InvalidCursorException` with `kind() == INVALID_CURSOR` and the
    fixed message.
11. **Implement missing-field branch** — after deserialization, null-
    check the three fields →
    `throw new InvalidCursorException(INVALID_CURSOR)`. Re-run —
    green.
12. **Failing test (DoD: invalid UUID / invalid timestamp / wrong fp
    length & non-hex chars).** Four parameterized cases:
    - `id` is `"not-a-uuid"`.
    - `rt` is `"2026-13-40T99:99:99"` (non-parseable).
    - `fp` is `"abc"` (wrong length).
    - `fp` is `"zzzzzzzzzzzzzzzz"` (correct length, non-hex).
    Each must throw `InvalidCursorException` with
    `kind() == INVALID_CURSOR` and the fixed message, and must NOT
    contain any substring of the input cursor's `fp`, `id`, or `rt`
    in the exception message.
13. **Implement strict type/format checks.** Use Jackson's typed
    deserialization (e.g. `@JsonProperty` on the record's components,
    or a small DTO) so invalid UUID / non-ISO timestamp raises
    `JsonProcessingException` and falls into the bad-JSON branch
    (re-thrown as `new InvalidCursorException(INVALID_CURSOR, cause)`).
    Add an explicit `fp` regex check `^[0-9a-f]{16}$` after parsing;
    on miss → `throw new InvalidCursorException(INVALID_CURSOR)`.
    Re-run — green.
14. **Failing test (DoD: wrong fingerprint → mismatch).** Encode a
    cursor with `fp = codec.fingerprintFor(filtersA)`. Decode with
    `expectedFingerprint = codec.fingerprintFor(filtersB)`. Assert
    `InvalidCursorException` is thrown with
    `kind() == CURSOR_FILTER_MISMATCH` and the fixed message
    `"cursor was issued for a different filter set"`. Run — currently
    no mismatch branch.
15. **Implement fingerprint check** — after structural validation,
    `if (!cursor.fp().equals(expectedFingerprint)) throw new
    InvalidCursorException(CURSOR_FILTER_MISMATCH);`. Re-run — green.
16. **Failing test (DoD: `fingerprintFor` public helper).** In
    `CursorCodecTest::fingerprintFor_matchesUtility`, assert that
    `codec.fingerprintFor(criteria)` returns the same 16-hex-char
    string for equal-by-value `QueryCriteria` instances and differs
    when any normalized filter changes. Run — fails (helper missing).
17. **Implement `fingerprintFor`** as a one-liner delegating to the
    package-private `FilterFingerprint.compute(criteria)`. Re-run —
    green.
18. **Failing test (DoD: no leak in any error path).** Add a single
    parameterized assertion across all error paths (both
    `INVALID_CURSOR` and `CURSOR_FILTER_MISMATCH`): the input `raw`
    cursor string and any single decoded field value must not appear
    as a substring of `e.getMessage()`. This is the explicit
    anti-leak guarantee from `tasks.md` T-6 DoD; the unified
    exception class makes it expressible in one parameterized
    assertion.
19. **Static checks.** `mvn spotless:apply` then run unit tests (do
    not invoke raw `pytest` — irrelevant, Java project; use
    `./mvnw test -pl . -Dtest=CursorCodecTest,FilterFingerprintTest`
    or the full `./mvnw test`). ArchUnit `LayeringTest` must stay
    green — the new package depends only on `domain/shared/Outcome`
    (via `QueryCriteria`) and `com.fasterxml.jackson.databind`. No
    Spring Web, no JPA.
20. **LOC budget check.** `git diff --stat main...HEAD`, excluding
    generated. Budget ~280 LOC; split if approaching 400 (split
    candidate: move `FilterFingerprint` to a separate task — only if
    triggered).

## Risks / open questions

- **Glossary alignment.** This plan adopts `glossary.md` as the
  source of truth for type names, packages, and exception shape.
  Specifically: `Cursor(rt, id, fp)`, `CursorCodec.{encode, decode,
  fingerprintFor}`, and a single `InvalidCursorException` with
  nested `ProblemKind { INVALID_CURSOR, CURSOR_FILTER_MISMATCH }`.
  If the glossary changes, this plan follows.
- **ISO-8601 string canonicalization across JDK versions.** Java's
  `OffsetDateTime.toString()` will emit `Z` for `ZoneOffset.UTC` and
  variable fractional-second precision (e.g. trailing zeros trimmed).
  Mitigation: pin the format via an explicit `DateTimeFormatter` and
  unit-test it against a hard-coded expected canonical string. The
  format must be **deterministic** — that is the load-bearing
  property, not the specific format choice. Decision locked in step
  4 implementation.
- **`QueryCriteria` field naming drift.** If T-5 names the
  `event_type` filter `eventType` (camelCase in Java) but the
  fingerprint must key on `"event_type"` (snake_case in the canonical
  JSON, matching the wire spec), the mapping is owned **here**, not
  in T-5. `FilterFingerprint` holds the canonical key names as
  package-private constants. If T-5 changes its field set after this
  task lands, this file must be updated alongside it (call out in PR
  description).
- **Jackson `ObjectMapper` allocation cost.** Mappers are expensive
  to construct but thread-safe to reuse. Codec accepts one via the
  constructor; tests build a private one; T-7 wires the production
  bean. No `static` mapper in this class to keep it
  injection-friendly.
- **No HMAC.** Explicit per `tasks.md` T-6 Notes and `design.md` §6.
  Carried as design-level open question §12. If security review
  forces HMAC pre-prod, the change is additive (new `sig` field; old
  cursors invalidated by version bump). Out of scope here.
