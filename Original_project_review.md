# Weakness and first step

**Weakness**
- Наличие PRD, которые "не очень": 
  - которые иногда протиречивы, не приоритизированы, достаточно абстрактны или наоборот, внутри указан чуть ли не формат, как надо хранить информацию(по моему мнению, PRD не должен ограничивать технически реализацию).
  - нету связей между PRD, иногда для разработки надо знать 2-4 PRD, но в них нету ссылок друг на друга (в последнее время изменилось и в новых документах появляюся ссылки)
  - Без указания, что уже сделано (или в каком виде есть).
- По командной разработке: 
  - задание общих принципов, чтобы не было разнородности в коде, единый подход к работе (specsmd/gsd/собственные skills)
  - явно разный context у разных разработчиков


**First steps**
- PRD:
  - Сделать linkage между PRD, чтобы подтягивалась инфа из других PRD (ну или чтобы стало понятно, куда смотреть).
  - Добавить скиллы для проверки PRD (особенно новых) на не противоречивость и полноту: если что-то вводится в PRD - оно должно иметь смысл(как по мне).
  - MCP для доступа к jira + skills для управления задачами (создание, обновление статуса, чтение и поиск).
- Командная разработка:
  - conventions
  - skills
  - rules
